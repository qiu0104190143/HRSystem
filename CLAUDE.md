# HR招聘管理系统 — 产品文档

> 本文档为HR招聘管理系统的完整产品说明，供产品、开发、运维人员参考。

## 一、产品概述

### 1.1 产品定位
面向中小企业的本地化招聘管理系统。运行在Windows电脑上，通过浏览器访问。支持多人协作（同局域网内），数据完全存储在本地，安全可靠。

### 1.2 核心价值
- **零成本部署**：无需购买服务器，本地电脑即可运行
- **多格式简历解析**：自动解析txt/Word/PDF简历
- **智能匹配**：5维度评分，自动推荐最合适的3位候选人
- **操作简便**：浏览器页面操作，无需技术背景

### 1.3 使用场景
- HR部门日常招聘管理
- 简历库的集中管理与搜索
- 职位发布与候选人智能匹配

## 三、匹配算法说明

### 3.1 五维度评分体系

| 维度 | 权重 | 评分方法 |
|------|------|---------|
| 学历匹配 | 20% | 学历等级 ≥ 要求等级得满分，低于要求按差距扣分 |
| 技能匹配 | 35% | 技能标签交集/并集比率，结合简历内容搜索 |
| 经验匹配 | 20% | 工作年限 ≥ 要求年限得满分，不足按比例扣分 |
| 年龄匹配 | 10% | 在设定范围内得满分，偏差按度扣分 |
| 内容相关性 | 15% | jieba分词 + Jaccard相似度计算 |

### 3.2 匹配度等级
- **≥80%**：🟢 强烈推荐
- **60%-79%**：🟡 可以考虑
- **<60%**：🔴 匹配较低

### 3.3 匹配原因示例
```
✅ 学历匹配：硕士，满足岗位要求
✅ 技能匹配度75%（3/4）。具备：Python，Java，MySQL；欠缺：Docker
✅ 经验匹配：5年工作经验，满足3年要求
✅ 年龄符合：32岁，在25-40范围内
⚠️ 内容相关性一般：简历内容与岗位描述部分匹配
```

## 四、技术架构

### 4.1 技术栈
| 层面 | 技术 | 版本 |
|------|------|------|
| 编程语言 | Python | 3.9+ |

## 六、常见问题

**Q: 如何让同事访问？**
A: 首页会显示本机局域网IP地址，同事在同一WiFi下浏览器访问 `http://你的IP:5000` 即可。

**Q: 数据存在哪里？**
A: 所有数据存储在 `database/hr.db`（SQLite数据库文件），上传的简历文件存储在 `uploads/resumes/` 目录。

**Q: 如何备份数据？**
A: 复制 `database/` 和 `uploads/` 两个文件夹即可完整备份。

**Q: 简历解析不准确怎么办？**
A: 上传后可在编辑页面手动修正所有结构化信息（姓名、学历、技能等）。
